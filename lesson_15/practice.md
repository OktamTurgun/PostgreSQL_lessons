## üéØ Mashg ºulot 1: INNER JOIN

### Vazifa 1.1
**Savol:** O ºquvchilar va ularning buyurtmalari ma ºlumotlarini ko ºrsating.

```sql
-- Yechim:
SELECT s.name, s.surname, o.order_date, o.total_amount
FROM students s
INNER JOIN orders o ON s.id = o.student_id;
```

### Vazifa 1.2
**Savol:** Mahsulotlar va ularning buyurtmalari ma ºlumotlarini ko ºrsating.

```sql
-- Yechim:
SELECT p.name, p.brand, o.quantity, o.total_amount
FROM products p
INNER JOIN orders o ON p.id = o.product_id;
```

### Vazifa 1.3
**Savol:** O ºquvchilar, mahsulotlar va buyurtmalar ma ºlumotlarini ko ºrsating.

```sql
-- Yechim:
SELECT s.name, s.surname, p.name as mahsulot, o.total_amount
FROM students s
INNER JOIN orders o ON s.id = o.student_id
INNER JOIN products p ON o.product_id = p.id;
```

### Vazifa 1.4
**Savol:** A balli o ºquvchilar va ularning buyurtmalari.

```sql
-- Yechim:
SELECT s.name, s.surname, o.order_date, o.total_amount
FROM students s
INNER JOIN orders o ON s.id = o.student_id
WHERE s.grade = 'A';
```

## üéØ Mashg ºulot 2: LEFT JOIN

### Vazifa 2.1
**Savol:** Barcha o ºquvchilar va ularning buyurtmalari (buyurtmasi yo ºq bo ºlsa ham).

```sql
-- Yechim:
SELECT s.name, s.surname, o.order_date, o.total_amount
FROM students s
LEFT JOIN orders o ON s.id = o.student_id;
```

### Vazifa 2.2
**Savol:** Barcha mahsulotlar va ularning sotilish ma ºlumotlari.

```sql
-- Yechim:
SELECT p.name, p.brand, COUNT(o.id) as sotilgan_soni
FROM products p
LEFT JOIN orders o ON p.id = o.product_id
GROUP BY p.id, p.name, p.brand;
```

### Vazifa 2.3
**Savol:** O ºquvchilar va ularning buyurtmalari (NULL bo ºlsa ham ko ºrsating).

```sql
-- Yechim:
SELECT s.name, s.surname, 
       CASE WHEN o.id IS NULL THEN 'Buyurtma yo ºq' 
            ELSE 'Buyurtma bor' END as holat
FROM students s
LEFT JOIN orders o ON s.id = o.student_id;
```

### Vazifa 2.4
**Savol:** Tashkentlik o ºquvchilar va ularning buyurtmalari.

```sql
-- Yechim:
SELECT s.name, s.surname, o.total_amount
FROM students s
LEFT JOIN orders o ON s.id = o.student_id
WHERE s.city = 'Tashkent';
```

## üéØ Mashg ºulot 3: RIGHT JOIN

### Vazifa 3.1
**Savol:** Barcha buyurtmalar va ularning o ºquvchilari.

```sql
-- Yechim:
SELECT o.order_date, o.total_amount, s.name, s.surname
FROM students s
RIGHT JOIN orders o ON s.id = o.student_id;
```

### Vazifa 3.2
**Savol:** Barcha buyurtmalar va mahsulotlar ma ºlumotlari.

```sql
-- Yechim:
SELECT o.order_date, p.name, p.brand, o.quantity
FROM products p
RIGHT JOIN orders o ON p.id = o.product_id;
```

### Vazifa 3.3
**Savol:** Buyurtmalar va ularning o ºquvchilari (sana bo ºyicha tartiblangan).

```sql
-- Yechim:
SELECT o.order_date, s.name, s.surname, o.total_amount
FROM students s
RIGHT JOIN orders o ON s.id = o.student_id
ORDER BY o.order_date;
```

```sql
FULL JOIN orders o ON p.id = o.product_id;
```

### Vazifa 4.3

## üéØ Mashg ºulot 5: CROSS JOIN

### Vazifa 5.1
**Savol:** Barcha o ºquvchilar va barcha mahsulotlar kombinatsiyasi.

```sql
-- Yechim:
SELECT s.name, s.surname, p.name as mahsulot, p.price
FROM students s
CROSS JOIN products p;
```

### Vazifa 5.2
**Savol:** Har bir o ºquvchi uchun har bir kategoriya.

```sql
-- Yechim:
SELECT s.name, c.name as kategoriya
FROM students s
CROSS JOIN categories c;
```

### Vazifa 5.3
**Savol:** O ºquvchilar va mahsulotlar (faqat 5 ta natija).

```sql
-- Yechim:
SELECT s.name, p.name as mahsulot
FROM students s
CROSS JOIN products p
LIMIT 5;
```

## üéØ Mashg ºulot 6: Murakkab JOIN so ºrovlar

### Vazifa 6.1
**Savol:** O ºquvchilar, ularning buyurtmalari va mahsulotlar to ºliq ma ºlumoti.

```sql
-- Yechim:
SELECT s.name, s.surname, p.name as mahsulot, 
       o.quantity, o.total_amount, o.order_date
FROM students s
INNER JOIN orders o ON s.id = o.student_id
INNER JOIN products p ON o.product_id = p.id
ORDER BY s.name, o.order_date;
```

### Vazifa 6.2
**Savol:** Har o ºquvchining buyurtmalari soni va umumiy summa.

```sql
-- Yechim:
SELECT s.name, s.surname, 
       COUNT(o.id) as buyurtmalar_soni,
       SUM(o.total_amount) as umumiy_summa
FROM students s
LEFT JOIN orders o ON s.id = o.student_id
GROUP BY s.id, s.name, s.surname
ORDER BY umumiy_summa DESC;
```

### Vazifa 6.3
**Savol:** Har mahsulotning nechta marta sotilgani va umumiy tushumi.

```sql
-- Yechim:
SELECT p.name, p.brand, 
       COUNT(o.id) as sotilgan_soni,
       SUM(o.total_amount) as umumiy_tushum
FROM products p
LEFT JOIN orders o ON p.id = o.product_id
GROUP BY p.id, p.name, p.brand
ORDER BY umumiy_tushum DESC;
```

### Vazifa 6.4
**Savol:** Har brenddagi mahsulotlar va ularning sotilish statistikasi.

```sql
-- Yechim:
SELECT p.brand,
       COUNT(DISTINCT p.id) as mahsulotlar_soni,
       COUNT(o.id) as sotilgan_soni,
       SUM(o.total_amount) as umumiy_tushum
FROM products p
LEFT JOIN orders o ON p.id = o.product_id
GROUP BY p.brand
ORDER BY umumiy_tushum DESC;
```

## üéØ Mashg ºulot 7: JOIN turlari taqqoslash

### Vazifa 7.1
**Savol:** INNER JOIN vs LEFT JOIN taqqoslash - o ºquvchilar va buyurtmalar.

```sql
-- INNER JOIN: faqat buyurtmasi bor o ºquvchilar
SELECT s.name, COUNT(o.id) as buyurtmalar_soni
FROM students s
INNER JOIN orders o ON s.id = o.student_id
GROUP BY s.id, s.name;

-- LEFT JOIN: barcha o ºquvchilar (buyurtmasi yo ºq bo ºlsa ham)
SELECT s.name, COUNT(o.id) as buyurtmalar_soni
FROM students s
LEFT JOIN orders o ON s.id = o.student_id
GROUP BY s.id, s.name;
```

### Vazifa 7.2
**Savol:** LEFT JOIN vs RIGHT JOIN taqqoslash.

```sql
-- LEFT JOIN: barcha o ºquvchilar
SELECT s.name, o.order_date
FROM students s
LEFT JOIN orders o ON s.id = o.student_id;

-- RIGHT JOIN: barcha buyurtmalar
SELECT s.name, o.order_date
FROM students s
RIGHT JOIN orders o ON s.id = o.student_id;
```

## üéØ Mashg ºulot 8: JOIN shartlari

### Vazifa 8.1
**Savol:** O ºquvchilar va ularning 2023-yilgi buyurtmalari.

```sql
-- Yechim:
SELECT s.name, s.surname, o.order_date, o.total_amount

INNER JOIN orders o ON s.id = o.student_id 
    AND o.order_date >= '2023-01-01'
```

### Vazifa 8.2
**Savol:** A balli o ºquvchilar va ularning qimmat buyurtmalari.

```sql
-- Yechim:
SELECT s.name, s.surname, o.total_amount
FROM students s
INNER JOIN orders o ON s.id = o.student_id
WHERE s.grade = 'A' AND o.total_amount > 1000000;
```

### Vazifa 8.3
**Savol:** Apple brendidagi mahsulotlar va ularning buyurtmalari.

```sql
-- Yechim:
SELECT p.name, o.quantity, o.total_amount
FROM products p
LEFT JOIN orders o ON p.id = o.product_id
WHERE p.brand = 'Apple';
```

## üéØ Qo ºshimcha vazifalar

### Vazifa 9.1
**Savol:** Har oy qaysi mahsulotlar sotilgan va ularning umumiy miqdori.

```sql
-- Yechim:
SELECT 
    EXTRACT(MONTH FROM o.order_date) as oy,
    p.name,
    COUNT(o.id) as sotilgan_soni,
    SUM(o.total_amount) as umumiy_summa
FROM products p
INNER JOIN orders o ON p.id = o.product_id
GROUP BY EXTRACT(MONTH FROM o.order_date), p.name
ORDER BY oy, umumiy_summa DESC;
```

### Vazifa 9.2
**Savol:** Har shahardagi o ºquvchilar va ularning buyurtmalari statistikasi.

```sql
-- Yechim:
SELECT s.city,
       COUNT(DISTINCT s.id) as oquvchilar_soni,
       COUNT(o.id) as buyurtmalar_soni,
       SUM(o.total_amount) as umumiy_summa
FROM students s
LEFT JOIN orders o ON s.id = o.student_id
GROUP BY s.city
ORDER BY umumiy_summa DESC;
```

### Vazifa 9.3
**Savol:** Har kategoriyadagi mahsulotlar va ularning sotilish ma ºlumotlari.

```sql
-- Yechim:
SELECT p.category,
       COUNT(DISTINCT p.id) as mahsulotlar_soni,
       COUNT(o.id) as sotilgan_soni,
       ROUND(AVG(p.price), 2) as o_rtacha_narx
FROM products p
LEFT JOIN orders o ON p.id = o.product_id
GROUP BY p.category
ORDER BY sotilgan_soni DESC;
```

## üìå Natijalarni tekshirish

Har bir vazifani bajarganingizdan so ºng, natijalarni tekshiring:

1. **INNER JOIN** ‚Äî faqat mos keladigan yozuvlar qaytarilganmi?
2. **LEFT JOIN** ‚Äî chap jadvaldagi barcha yozuvlar qaytarilganmi?
3. **RIGHT JOIN** ‚Äî o ºng jadvaldagi barcha yozuvlar qaytarilganmi?
4. **FULL JOIN** ‚Äî ikkala jadvaldagi barcha yozuvlar qaytarilganmi?
5. **CROSS JOIN** ‚Äî barcha kombinatsiyalar qaytarilganmi?
